8.	API ROUTES 
8.1 POST /api/literature
Input:
{
  hypothesis: string
}
Behavior:
1.	Validate input. 
2.	Parse hypothesis into structured fields. 
3.	Generate literature search queries. 
4.	Query Semantic Scholar. 
5.	Optionally query arXiv if hypothesis domain suggests physics, computation, materials, AI, or preprints. 
6.	Optionally query PubMed/OpenAlex/Crossref if easy and useful. 
7.	Use Tavily for protocol repository search if TAVILY_API_KEY is present. 
8.	Deduplicate references. 
9.	Ask OpenAI to classify novelty using retrieved references. 
10.	Return strict LiteratureQCSchema. 
11.	If OpenAI key missing but search works, return deterministic heuristic QC. 
12.	If all external APIs missing and demo fallback enabled, return clearly marked demo fallback result. 
13.	Never hallucinate references. 
Novelty classification guidance:
•	exact_match_found: same organism/system, same intervention, same comparator/control, same primary outcome, same or close quantitative target, clear prior protocol/paper. 
•	similar_work_exists: related method or intervention exists, but differs in system, endpoint, threshold, material, or operational details. 
•	not_found: no close match found in available search results. 
•	If coverage is weak, include coverage warning and lower confidence. 
Output:
LiteratureQCSchema

12.1 Retrieval inputs

12.2 Retrieval scoring
Score every stored feedback item.
Use a transparent scoring formula.
Suggested formula without embeddings:
score =
  0.25 * domainMatch
+ 0.25 * experimentTypeMatch
+ 0.20 * tagJaccard
+ 0.15 * keywordOverlap
+ 0.10 * applicabilityBoost
+ 0.05 * severityConfidenceBoost
Definitions:
•	domainMatch = 1 if same domain, 0.5 if related, 0 otherwise 
•	experimentTypeMatch = 1 if same/similar experiment type, 0 otherwise 
•	tagJaccard = intersection(tags) / union(tags) 
•	keywordOverlap = overlap of normalized content words 
•	applicabilityBoost: 
o	broad_rule: 1 
o	similar_experiment_type: 0.8 
o	only_this_plan: 0.2 unless exact same source_plan/hypothesis 
•	severityConfidenceBoost combines severity and feedback confidence 
If embeddings are available:
•	compute embedding for feedback derived rule + tags + domain + experiment type 
•	compute embedding for new hypothesis 
•	blend: 
o	0.60 embedding cosine similarity 
o	0.40 lexical/domain score 
Use text-embedding-3-small if available, but make it optional.
Do not fail plan generation if embeddings fail.
12.3 Retrieval output
Return top 3–7 feedback items.
Filter out low relevance:
•	default minimum score: 0.25 
•	always include critical broad rules if score > 0.2 
12.4 Handling contradictions
If two feedback rules conflict:
•	include both in applied feedback with warning 
•	instruct model to be conservative 
•	prefer: 
1.	critical severity 
2.	higher confidence 
3.	more recent 
4.	more specific domain/experiment match

29.1 Literature route pseudo-flow
export async function POST(req: Request) {
  try {
    const body = LiteratureRequestSchema.parse(await req.json());

    const parsed = await parseHypothesis(body.hypothesis);

    const queries = generateLiteratureQueries(parsed, body.hypothesis);

    const semanticResults = await searchSemanticScholar(queries);
    const arxivResults = shouldUseArxiv(parsed)
      ? await searchArxiv(queries)
      : [];
    const protocolResults = await searchProtocolRepositories(queries);

    const references = dedupeReferences([
      ...semanticResults,
      ...arxivResults,
      ...protocolResults
    ]);

    const qc = await classifyNovelty({
      hypothesis: body.hypothesis,
      parsed,
      references,
      queries
    });

    return Response.json(LiteratureQCSchema.parse(qc));
  } catch (err) {
    return jsonError(err);
  }
}
29.2 Generate-plan route pseudo-flow
export async function POST(req: Request) {
  try {
    const body = GeneratePlanRequestSchema.parse(await req.json());

    const parsed = body.literature_qc.parsed_hypothesis;

    const feedback = await retrieveRelevantFeedback({
      hypothesis: body.hypothesis,
      parsed_hypothesis: parsed,
      limit: 7
    });

    const protocolEvidence = await searchProtocols(body.hypothesis, parsed);
    const supplierEvidence = await searchSuppliers(body.hypothesis, parsed);

    const evidenceCards = mergeEvidenceCards(protocolEvidence, supplierEvidence);

    const plan = await generateExperimentPlan({
      hypothesis: body.hypothesis,
      parsed,
      literatureQC: body.literature_qc,
      evidenceCards,
      feedback
    });

    return Response.json(ExperimentPlanSchema.parse(plan));
  } catch (err) {
    return jsonError(err);
  }
}
29.3 Feedback route pseudo-flow
export async function POST(req: Request) {
  try {
    const body = FeedbackCreateRequestSchema.parse(await req.json());

    const derived = await deriveFeedbackRule(body);

    const item = ScientistFeedbackSchema.parse({
      id: crypto.randomUUID(),
      created_at: new Date().toISOString(),
      ...body,
      derived_rule: derived.derived_rule,
      tags: derived.normalized_tags.length ? derived.normalized_tags : body.tags
    });

    await appendFeedback(item);

    return Response.json({ ok: true, feedback: item });
  } catch (err) {
    return jsonError(err);
  }
}
